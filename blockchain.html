Relavent experience for blockchain / haskell

Some of the things I've done relevant to blockchain and / or haskell.

Hath

Hath is a suite of utlities created while I was working with Komodo. Of interest:

  • Consensus protocol - A consensus protocol, using distributed-haskell for networking, and using multiple rounds and authenticated votes similar to Tendermint. This is not designed to be BFT, it's designed to enable a distributed gateway for posting merkle proofs between blockchains.
  • A cross chain notariser using above consensus.
  • Patricia Merkle tree in Haskell, and tests. Also, the Ethereum transaction.

Crypto-Conditions

Crypto-Conditions is a composite challenge / response protocol for cryptographic ownership, similar in scope to Miniscript for Bitcoin. It defines a merkle tree of n-of-m cryptographic challenges. BigchainDB used this while I worked there, and I implemented it in Haskell. Later, I was hired by Komodo to implement it in C.
